# iOS Simulator Control

Recipes for controlling the iOS simulator from zsh. Use these the same way you'd use Chrome DevTools MCP on the web side: tight loops of "make change → run → screenshot → verify".

The iOS app is generated by **Tuist** from `apps/ios/Project.swift` — `.xcodeproj` is not committed. All build/install/screenshot operations have a `mise` task in `apps/ios/mise.toml`. Prefer those; the raw commands below exist for cases where you need to deviate.

## Prerequisites

- Xcode + Command Line Tools installed (`xcode-select --install`).
- `mise install` run inside `apps/ios/` (installs Tuist and `swift-format`).
- `idb` (Facebook's iOS bridge) installed: `brew tap facebook/fb && brew install idb-companion && pip install fb-idb`.
- One simulator booted as the working device.

## The fast path (mise tasks)

```sh
mise run -C apps/ios sim:list             # list available simulators
mise run -C apps/ios sim:boot             # boot the configured simulator + open Simulator.app
mise run -C apps/ios sim:launch           # build + install + launch (depends on sim:install + build)
mise run -C apps/ios sim:screenshot       # screenshot booted simulator to apps/ios/.build/simulator/$APP_SCHEME.png
mise run -C apps/ios sim:erase            # wipe the simulator's data
```

Configure via env vars in `apps/ios/mise.toml` (or override per-invocation):

```sh
SIMULATOR_NAME="iPhone 16 Pro" mise run -C apps/ios sim:launch
```

## Raw recipes (only when you need to deviate)

### List simulators

```sh
xcrun simctl list devices available
```

Find a runtime + device pair you want and grab its UDID.

### Boot a simulator

```sh
# Boot the named device
xcrun simctl boot "iPhone 16"

# Boot by UDID
xcrun simctl boot 12345678-1234-1234-1234-123456789ABC

# Open Simulator.app so you can see it
open -a Simulator
```

### Build and install the app

Prefer `mise run -C apps/ios sim:install`. Under the hood:

```sh
# Generate the Xcode project (Tuist), then build for the configured simulator.
# These two commands are what `mise run -C apps/ios build` runs.
cd apps/ios && tuist generate --no-open
cd apps/ios && xcodebuild \
  -project App.xcodeproj \
  -scheme App \
  -destination 'platform=iOS Simulator,name=iPhone 16' \
  -derivedDataPath .derived-data \
  build

# Install the built app onto the booted simulator
xcrun simctl install booted apps/ios/.derived-data/Build/Products/Debug-iphonesimulator/App.app

# Launch
xcrun simctl launch booted com.sdd.items
```

### Screenshots

```sh
# Save a PNG of the current screen (one-shot)
xcrun simctl io booted screenshot /tmp/sim-screenshot.png

# Then use the Read tool on /tmp/sim-screenshot.png to view it
```

Or use the mise task, which writes to a stable location under `.build/simulator/`:

```sh
mise run -C apps/ios sim:screenshot
```

Take screenshots aggressively when verifying visual changes. Compare before/after by saving with descriptive names.

### Tap, type, swipe (via idb)

`idb` provides higher-level interaction than `simctl`:

```sh
# Tap at point (x, y) — coordinates are in points, top-left origin
idb ui tap --udid <udid> 100 200

# Type text
idb ui text --udid <udid> "Hello"

# Swipe from (x1, y1) to (x2, y2)
idb ui swipe --udid <udid> 100 500 100 100

# Press a hardware button
idb ui button --udid <udid> HOME

# Get the UI hierarchy as JSON (great for finding accessibility identifiers)
idb ui describe-all --udid <udid>
```

To get the booted simulator's UDID:

```sh
booted_udid=$(xcrun simctl list devices booted | grep -oE '[A-F0-9-]{36}' | head -1)
```

### Logs

```sh
# Stream the simulator's system log, filtered for our app's bundle id
xcrun simctl spawn booted log stream --predicate 'subsystem == "com.sdd.items"'

# One-shot log dump (last 5 minutes)
xcrun simctl spawn booted log show --last 5m --predicate 'subsystem == "com.sdd.items"'
```

For a tight verify loop, use the streaming form in a backgrounded `Bash` call (`run_in_background: true`) and grep its output as you interact.

### Reset state

```sh
# Wipe the simulator's data (fresh state, same simulator) — or use `mise run sim:erase`
xcrun simctl erase booted
```

## Verify-iterate loop pattern

```sh
# 1. Make a code change

# 2. Build + install + launch in one shot
mise run -C apps/ios sim:launch

# 3. Drive the UI to the state you want to verify
idb ui tap --udid <udid> ...

# 4. Screenshot
mise run -C apps/ios sim:screenshot

# 5. Read apps/ios/.build/simulator/App.png to verify

# 6. Repeat
```

If you find yourself running this sequence three times, wire up a higher-level task in `apps/ios/mise.toml`.

## When NOT to use this skill

- Pure unit tests — run those via `mise run -C apps/ios t` instead.
- One-off lookups of simulator state that aren't tied to verifying a code change.
- Anything that has a faster path through Swift Testing or Xcode's test plan.
